▼m[CN=1005_ステータス] // ステータスバッジ（＝シナプス強度）
```
▼m[CN=名前] // コメント [⊕0+0]    ← 展開デフォルト
▶m[CN=名前] // コメント [⊖0+0]    ← 折畳デフォルト
▼m[CN=名前] // コメント [⊘0+0]    ← ロック（クリック無効）
▼m[CN=名前] // コメント [∞]       ← カウントしない
▼m[CN=名前] // コメント [∞10+4]   ← 10+10^4=10,010 で停止済み
▼m[CN=名前] // コメント [5]        ← 簡略形（⊕5+0 と同等）
```

| 記号 | 意味 |
|------|------|
| `⊕` | 展開状態 |
| `⊖` | 折畳状態 |
| `⊘` | ロック（操作不可） |
| `∞` | カウントしない（無制限） |
| `N+M` | カウント値 = N + 10^M（コンパクト表現） |
| `∞N+M` | N + 10^M でカウント停止済み |

クリックして膜を展開するたびにカウントが自動インクリメントされ、Markdownソースに書き戻される。

**カウント数 = シナプス強度（重み）**

カウント数は単なる開閉回数ではなく、**その膜が思考回路でどれだけ活性化されてきたか**を示す。

| 神経科学の概念 | markMupの対応 |
|---|---|
| シナプス強度 | 展開カウント数 N |
| ヘッブ学習（発火→強化） | 開くたびにNが増える |
| 事前学習済み重み | 最初からNに任意の値を設定可能 |
| 未発達シナプス / デッドコード | カウントゼロ `[⊕0+0]` |

**デフォルト値に学習済み強度を書き込む**ことで、文書を開いた瞬間に知識の習熟度マップが見える：
```
▼m[CN=3421_重要概念] [⊕80+0]   ← 既に80回学習済み
▼m[CN=7823_周辺知識] [⊕12+0]   ← 比較的弱い結合
▼m[CN=1205_新概念]   [⊕1+0]    ← まだ1回しか発火していない
```

**Me記法（膜間リンク）**

送り手と受け手それぞれを `{}` で囲み、`⇒Me⇒` でつなぐ：

```
{結合元} ⇒Me⇒ {結合先}
例: {1234_A} ⇒Me⇒ {5678_B, 9012_C}
```

開始膜の直下に書く場合、自膜が結合元のときは略記できる：

```
▼m[CN=1234_A] // Aの説明
// ⇄ {5678_B, 9012_C}    ← 双方向リンクの略記

内容
▲m[CN=1234_A]
```

- `⇄` = 双方向結合。`⇒Me⇒` = 単方向（送り手→受け手）
- `{}` でパース境界が明確 → `\{([^}]+)\} ⇒Me⇒ \{([^}]+)\}` で抽出可能
- `grep "5678"` で結合先に即ジャンプ（特別なツール不要）

---

**`{}` の中に書けるもの**

| 種類 | 記法例 | 接続先 |
|---|---|---|
| 内部膜リンク | `{3421_概念名}` | 同文書内の膜 |
| **外部URL** | `{https://example.com}` | **インターネット全体** |
| 他ノートの膜（将来） | `{ノートID::CN番号}` | Joplin内の別ノート |

内部リンクと外部URLは**混在可能**：
```
▼m[CN=3421_量子コンピュータ] [⊕28+0] // 学習中の概念
// ⇄ {7823_関連概念, 1205_応用例}
// ⇄ {https://arxiv.org/abs/xxxx, https://nature.com/articles/xxx}
▲m[CN=3421_量子コンピュータ]
```

外部URLにも膜の「強度・状態・意味」が付き、単なるブックマークを超える。

**参照膜（中身なし・リンクのみ）**

膜の中身を書かず、リンク先だけを持つ「参照膜」も有効：

```
▼m[CN=REF_概念名] [⊕12+0] // 参照の理由
// ⇄ {3421_概念名}
▲m[CN=REF_概念名]
```

これは **BTRON仮身・HTMLリンクの上位互換**。  
リンクのみだが、膜として「強度・状態・意味・双方向性」を持つ生きたポインタになる。

| | aタグ / 仮身 | markMup参照膜 |
|---|---|---|
| リンク先 | ○ | ○ |

| 今も有効か | ✗ | ○（展開/折畳み） |
| 他との関係 | ✗ | ○（⇄で並記） |

```

```

```
CN=NNNN_名前

```

**設計思想：**
- `NNNN` = ランダム4桁数字（名前との合成キーで衝突リスクを排除）
- `名前` = あらゆる言語・文字が使用可能（日本語・英語・中国語・アラビア語・絵文字・記号、異言語の組み合わせも自由）
- `grep "5831"` → 巨大ソースコードから1件だけヒット確定
- 4桁 × Unicode全文字 × 多言語組み合わせ = 無限のそのまた無限の固有値空間

**💡 NNNN 簡易生成法：その瞬間の「分秒」を使う**
```
10:38 に作成 → CN=1038_名前
11:04 に作成 → CN=1104_名前
```
時計を見てそのまま打つだけ。同じ分秒に同じ名前の膜を作らない限り衝突しない。

**メリット：**
- 数字だけで検索 → 部分一致なし、1件確定
- セッション間引き継ぎの「コードの住所」
- AIとの協働効率を劇的に向上（体感100倍以上）

| バージョン | 記法 | 備考 |
|-----------|------|------|
| v1.x | `\▼[CN=...]` | バックスラッシュ記法（初期） |

markMupの膜は **細胞モデル・脳モデル・情報モデルの統一体** である。

**構造の対応：**

| 生命の構造 | markMupの対応 |
|---|---|
| 細胞全体 | ノート全体 |
| 細胞膜（外壁） | 最外殻の膜 |
| ミトコンドリア | 処理系の入れ子膜 |
| 核 | 定義・設計図を持つ入れ子膜 |
| DNA（核の内容） | 膜の中に書かれたテキスト |
| ヘテロクロマチン（不活性） | 折畳み ⊖（休眠） |
| ユークロマチン（活性） | 展開 ⊕（活動） |
| 細胞膜の受容体・チャネル | 膜の名前・コメント |
| 臓器間メッセージ伝達経路 | Me記法リンク（⇄） |
| シナプス強度（重み） | 展開カウント数 |

**膜の2層構造：**
- **静的構造（解剖学）** = 膜の入れ子 → 臓器の配置・細胞の内部構造
- **動的関係（生理学）** = Me記法リンク → 臓器間の信号・代謝経路

この2層が揃って初めて、**生きたシステム**になる。

**普遍性：**

膜モデルは文書だけでなく、あらゆる情報構造に適用できる：

| 対象 | 膜の内容 | カウント | リンク |
|---|---|---|---|
| 文書・ノート | 知識・概念 | 参照頻度 | 概念間の関係 |
| プログラムコード | 関数・モジュール | 呼び出し頻度 | 依存関係 |
| AIの学習データ | 訓練サンプル | 学習回数・重み | データの関係性 |

**既存システムとの関係：**
```
aタグ (1991)            → 「指差し」のみ
BTRON 仮身 (1987〜)     → 「実身への分身」── 膜ではない
Ted Nelson Xanadu       → 「トランスクルージョン」── 重みがない
markMup膜 + Meリンク    → 「意味・強度・状態・伝達経路を持つ生きたリンク」
```

BTRONが夢見た **実身と仮身を同じ文法で書けること** を、  
markMupは **膜という1つの概念で統一した**。
