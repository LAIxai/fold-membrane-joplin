\▼[CN=1001_位置づけ] // 位置づけ
**markMup は新しい言語ではない。**

- HTML / Markdown / プレーンテキスト
- コード（JS / CSS / TypeScript / 他）
- AIメモリファイル

👉 **その上に"膜"を重ねる仕様**

- 表示系・保存形式・レンダラは一切変更しない
- **人間にもAIにも同時に効く構造境界**
- 実装に縛られない「検証仕様」として位置づける
\▲[CN=1001_位置づけ]

\▼[CN=1005_ステータス] // ステータスバッジ
```
\▼[CN=名前] // コメント [⊕0+0]    ← 展開デフォルト
\▶[CN=名前] // コメント [⊖0+0]    ← 折畳デフォルト
\▼[CN=名前] // コメント [⊘0+0]    ← ロック（クリック無効）
\▼[CN=名前] // コメント [∞]       ← カウントしない
\▼[CN=名前] // コメント [∞10+4]   ← 10+10^4=10,010 で停止済み
\▼[CN=名前] // コメント [5]        ← 簡略形（⊕5+0 と同等）
```

| 記号 | 意味 |
|------|------|
| `⊕` | 展開状態 |
| `⊖` | 折畳状態 |
| `⊘` | ロック（操作不可） |
| `∞` | カウントしない（無制限） |
| `N+M` | カウント値 = N + 10^M（コンパクト表現） |
| `∞N+M` | N + 10^M でカウント停止済み |

クリックするたびにカウントが自動インクリメントされ、Markdownソースに書き戻される。
\▲[CN=1005_ステータス]

\▼[CN=1008_相互参照] // 相互参照記法 v2.1（NEW）
// ⇄ {1009_固有番号}

開始膜の直後の行に `// ⇄ {}` で関連する膜を列挙する：

```
\▼[CN=1234_A] // Aの説明
// ⇄ {5678_B, 9012_C}

内容
\▲[CN=1234_A]
```

**設計思想：**
- `⇄` = 双方向リンク
- `{}` で囲む = ツール・AIがパース可能な構造化リスト
- `grep "5678"` で参照先に即ジャンプ
- 目次に `⇄ {NNNN_name}` を併記 → 目次だけでリンクを辿れる
- 特別な仕組み不要。実身/仮身相当をプレーンテキストで実現
- どんなエディタ・AI・ツールでも等しく使える

**目次での記法：**
```
\▼[CN=0001_TOC]
  \▼[CN=1234_A]  ⇄ {5678_B, 9012_C}
  \▼[CN=5678_B]  ⇄ {1234_A}
\▲[CN=0001_TOC]
```
\▲[CN=1008_相互参照]

\▼[CN=1009_固有番号] // CN=固有番号+名前 命名規則（実装拡張）
// ⇄ {1008_相互参照, 1010_AIメモリ}

```
CN=NNNN_名前
例: CN=5831_FILE_HEADER
    CN=3658_insertTemplate
    CN=4471_mupInsertBookmark
```

**設計思想：**
- `NNNN` = ランダム4桁数字（名前との合成キーで衝突リスクを排除）
- `名前` = 人間が読める説明（日本語可）
- `grep "5831"` → 巨大ソースコードから1件だけヒット確定
- 4桁+名前の組合せ = 実質無限の固有値空間

**メリット：**
- 数字だけで検索 → 部分一致なし、1件確定
- セッション間引き継ぎの「コードの住所」
- AIとの協働効率を劇的に向上（体感100倍以上）
\▲[CN=1009_固有番号]

\▼[CN=1010_AIメモリ] // 膜構造AIメモリ（世界初）
// ⇄ {1009_固有番号}

MEMORY.md を膜記法で構造化することで、AIのセッション間記憶を実現する：

```
\▼[CN=3001_20260323] // 2026.03.23(月)
  \▼[CN=3001_A] // バージョン履歴
  ...
  \▲[CN=3001_A]
\▲[CN=3001_20260323]
```

**効果：**
- セッション開始時にMEMORY.mdを自動読込 → 前回の続きから即開始
- CN番号でgrepすれば特定セクションに直行
- 過去の失敗・知見が蓄積 → 同じミスを繰り返さない
- 体感性能差：100倍以上（2026.03.23実証）

**運用：**
- `~/.claude/CLAUDE.md` に常設指示として記述
- 「昨日何をやった?」→ MEMORY.mdの日付セクションを即答
