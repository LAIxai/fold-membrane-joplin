▼m[CN=1005_ステータス] // ステータスバッジ（＝シナプス強度）
```
▼m[CN=名前] // コメント [⊕0+0]    ← 展開デフォルト
▶m[CN=名前] // コメント [⊖0+0]    ← 折畳デフォルト

▼m[CN=名前] // コメント [∞]       ← カウントしない
▼m[CN=名前] // コメント [∞10+4]   ← 10+10^4=10,010 で停止済み
▼m[CN=名前] // コメント [5]        ← 簡略形（⊕5+0 と同等）
```

| 記号 | 意味 |
|------|------|
| `⊕` | 展開状態 |
| `⊖` | 折畳状態 |
| `⊘` | ロック（操作不可） |
| `∞` | カウントしない（無制限） |
| `N+M` | カウント値 = N + 10^M（コンパクト表現） |
| `∞N+M` | N + 10^M でカウント停止済み |

クリックして膜を展開するたびにカウントが自動インクリメントされ、Markdownソースに書き戻される。

**折畳み表示の全7状態（中身・リンクの組み合わせ）**

折畳んだままで膜の状態が読める。🛒=中身なし、⇄=相互リンク、⇒=Me結合：

| # | 中身 | リンク | 折畳み表示 | 典型的な用途 |
|---|---|---|---|---|
| 1 | なし | なし | `[🛒]` | 空の予定枠・未着手 |
| 2 | なし | ⇄のみ | `⇄[🛒]` | 参照膜（リンクのみ） |
| 3 | なし | ⇒のみ | `⇒[🛒]` | Me結合のみの中継点 |
| 4 | あり | なし | `[⊖N+0]` | 通常の実体膜 |
| 5 | あり | ⇄のみ | `[⊖N+0] ⇄` | 中身あり＋相互参照 |
| 6 | あり | ⇒のみ | `[⊖N+0] ⇒` | 中身あり＋Me結合 |
| 7 | あり | ⇄と⇒ | `[⊖N+0] ⇄⇒` | 中身あり＋両方 |

```
▼▲m[CN=昼食_1200]         [🛒]           ← ①空の予定枠
▼▲m[CN=参考_260408]       ⇄[🛒]          ← ②相互リンクのみ
▼▲m[CN=処理フロー_0900]   ⇒[🛒]          ← ③Me結合のみ
▶m[CN=会議メモ_1400]      [⊖5+0]         ← ④中身あり
▶m[CN=設計書_1000]        [⊖12+0] ⇄      ← ⑤中身＋相互リンク
▶m[CN=データ変換_0830]    [⊖3+0] ⇒       ← ⑥中身＋Me結合
▶m[CN=基幹処理_0800]      [⊖8+0] ⇄⇒     ← ⑦中身＋両方
```

**読み方のルール：**
- `🛒` が見える → 中身がない
- `🛒` が見えない → 中身がある（カウントが証拠）
- `⇄` が見える → 相互リンクがある
- `⇒` が見える → Me結合がある

**カウント数 = シナプス強度（重み）**

カウント数は単なる開閉回数ではなく、**その膜が思考回路でどれだけ活性化されてきたか**を示す。

| 神経科学の概念 | markMupの対応 |
|---|---|
| シナプス強度 | 展開カウント数 N |
| ヘッブ学習（発火→強化） | 開くたびにNが増える |
| 事前学習済み重み | 最初からNに任意の値を設定可能 |
| 未発達シナプス / デッドコード | カウントゼロ `[⊕0+0]` |

**2つのモード：**
```
[⊕N+0]  ← カウンターモード（自動）: 展開するたびNが増える。N≥1なら書いた証拠。
[∞N]    ← 重要度モード（静的）: ∞が付くとカウント停止。Nは重要度の値。
```

**デフォルト値に学習済み強度を書き込む**ことで、文書を開いた瞬間に知識の習熟度マップが見える：
```
▼m[CN=3421_重要概念] [⊕80+0]   ← 既に80回学習済み
▼m[CN=7823_周辺知識] [⊕12+0]   ← 比較的弱い結合
▼m[CN=1205_新概念]   [⊕1+0]    ← まだ1回しか発火していない
```

**Me記法（膜間リンク）**

送り手と受け手それぞれを `{}` で囲み、`⇒Me⇒` でつなぐ：

```
{結合元} ⇒Me⇒ {結合先}
例: {1234_A} ⇒Me⇒ {5678_B, 9012_C}
```

開始膜の直下に書く場合、自膜が結合元のときは略記できる：

```
▼m[CN=1234_A] // Aの説明
// ⇄ {5678_B, 9012_C}    ← 双方向リンクの略記

内容
▲m[CN=1234_A]
```

- `⇄` = 双方向結合。`⇒Me⇒` = 単方向（送り手→受け手）
- `{}` でパース境界が明確 → `\{([^}]+)\} ⇒Me⇒ \{([^}]+)\}` で抽出可能
- `grep "5678"` で結合先に即ジャンプ（特別なツール不要）

---

**`{}` の中に書けるもの**

| 種類 | 記法例 | 接続先 |
|---|---|---|
| 内部膜リンク | `{3421_概念名}` | 同文書内の膜 |
| **外部URL** | `{https://example.com}` | **インターネット全体** |
| 他ノートの膜（将来） | `{ノートID::CN番号}` | Joplin内の別ノート |

内部リンクと外部URLは**混在可能**：
```
▼m[CN=3421_量子コンピュータ] [⊕28+0] // 学習中の概念
// ⇄ {7823_関連概念, 1205_応用例}
// ⇄ {https://arxiv.org/abs/xxxx, https://nature.com/articles/xxx}
▲m[CN=3421_量子コンピュータ]
```

外部URLにも膜の「強度・状態・意味」が付き、単なるブックマークを超える。

| | `NNNN_名前` | `名前_NNNN` |

**設計思想：**
- `NNNN` = 4桁数字（名前との合成キーで衝突リスクを排除）
- `名前` = あらゆる言語・文字が使用可能（日本語・英語・絵文字・記号・異言語混在も自由）
- `grep "5831"` → 巨大ソースコードから1件だけヒット確定
- 4桁 × Unicode全文字 × 多言語組み合わせ = 無限のそのまた無限の固有値空間

**💡 NNNN 簡易生成法：その瞬間の「分秒」を使う**
```
10:38 に作成 → CN=1038_名前  または  CN=名前_1038
11:04 に作成 → CN=1104_名前  または  CN=名前_1104
```
時計を見てそのまま打つだけ。同じ分秒に同じ名前の膜を作らない限り衝突しない。

**💡 日記・スケジュール用途（人間仕様の真骨頂）**

```

▼m[CN=散歩_0830]

▲m[CN=散歩_0830]
```

| 検索クエリ | わかること |
|---|---|
| `▼m[CN=朝食_` | 全日付の朝食開始時刻一覧 → 何時に食べることが多いか一目瞭然 |
| `▼m[CN=散歩_` | 散歩の頻度・時刻の分布 |
| `▼m[CN=起床_` | 起床時刻の変化 |

毎日の行動は名前が同じで時刻だけ変わるため、`名前_NNNN` 形式が自然にフィットする。

▼m[CN=1015_WYSIWYG操作] // WYSIWYGでの膜操作（v4.5 NEW）

WYSIWYGモードでは、膜の各部位が異なる役割を持つ。

---

**名前部分のプロテクション**

膜の名前（`▼m[CN=名前]` の `名前` 部分）は WYSIWYG では**編集不可**：

| 部位 | カーソル | 編集 | 理由 |
|------|---------|------|------|
| `▼` アイコン | 矢印 | ✗ | 開閉専用 |
| `CN=名前` 名前span | 矢印 | ✗ | 開始膜・閉じ膜の同期が必要 |
| `// コメント` 部分 | テキスト | ✓ | span化しない・安全 |
| `[⊕N+0]` バッジ | テキスト | ✓ | カウンター手動編集可 |

**名前を変更するにはMarkdownモードで行う。** WYSIWYGで名前を直接書き換えると、開始膜と閉じ膜の名前が不一致になり構造が壊れるため。

**カーソルナビゲーションの挙動（意図的な設計）：**
```
膜の前の行からコメントへ → 名前spanを飛び越えてコメントに着地
コメントの先頭から← → 名前spanの壁に激突・停止（上には出られない）
閉じ膜から→          → 膜の外（次の行）へ出られる
Cmd+↑               → 飛び越えてどこへでも移動可能
```

「上の壁・下の出口」という設計で、**コメントを編集中に矢印キーで膜の外へ飛び出してしまう無駄な操作を防ぐ**。  
矢印キーの移動範囲がコメント内に半ば閉じ込められることで、膜の中に集中して編集できる感覚が生まれる。  
どうしても外に出たい場合は `Cmd+↑` で飛び越えられる。

---

**右クリックメニュー（見えない栞ボタン）**

WYSIWYGで `▼` アイコンまたは名前spanを**右クリック**すると：

```
┌─────────────────────────────┐
│ ⇄ エディタ切替（名前の編集）  │
│ ─────────────────────────── │
│ 📋 名前をコピー               │
└─────────────────────────────┘
```

- **⇄ エディタ切替（名前の編集）**: Markdown ↔ WYSIWYG を即切替。名前を編集したい場合はここからMarkdownモードへ。
- **📋 名前をコピー**: 膜のCN名をクリップボードにコピー。相互参照記法 `{}` に貼り付けるときに便利。

Markdownプレビュー（右ペイン）でも同様のメニューが表示されるが、ラベルは「⇄ エディタ切替」（名前の編集なし）。

**膜は「見えない栞ボタン」を兼ねる：**

🔖しおりボタンが「明示的な位置マーカー」であるのに対し、  
膜の右クリックメニューは「どの膜からでもエディタを切り替えられる」暗黙のUI。

```
🔖m[ここだよ] → 明示的な栞（場所を示す）
▼m[CN=...] → 暗黙の栞（どこからでも切り替えられる）
```

膜が**折畳み・カーソルナビ・エディタ切替・名前コピー**の四役を兼ねる。このような多機能な膜UIは他に存在しない。

---

**インポーチン・エクスポーチン（生物学との対応）**

細胞の核には**核膜孔複合体（Nuclear Pore Complex）**という構造が存在する。  
核膜の孔はただの穴ではなく、何を通すかを選別するタンパク質の集合体である。

| タンパク質 | 役割 | markMupの対応 |
|---|---|---|
| **インポーチン（importin）** | 核の「外→中」への輸送を担う。決まった方向にしか通さない。 | ↓キー（膜の上から内部へ入る） |
| **エクスポーチン（exportin）** | 核の「中→外」への輸送を担う。同じく一方向のみ。 | →キー（閉じ膜から外へ出る） |
| **通さない方向** | 逆向きの輸送は行わない | ↑キー・←キー（名前spanの壁） |
| **ATP駆動ポンプ（緊急）** | エネルギーを使って強制的に輸送する特殊経路 | Cmd+↑（壁を強制突破） |

科学番組では「通す・通さない」という概念だけが語られることが多いが、  
実際には**方向が決まったタンパク質の鍵**がなければ通れない。

markMupの膜における↓と→の「鍵」は、まさにインポーチン・エクスポーチンと同じ設計思想——  
**方向性を持った選択的透過境界**——を、テキストエディタ上で実現したものである。
