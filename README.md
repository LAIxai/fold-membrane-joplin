▼_M[CN=1005_ステータス] // ステータスバッジ
```
$▼_M[CN=名前]$ // コメント [⊕0+0]    ← 展開デフォルト
$▶_M[CN=名前]$ // コメント [⊖0+0]    ← 折畳デフォルト
$▼_M[CN=名前]$ // コメント [⊘0+0]    ← ロック（クリック無効）
$▼_M[CN=名前]$ // コメント [∞]       ← カウントしない
$▼_M[CN=名前]$ // コメント [∞10+4]   ← 10+10^4=10,010 で停止済み
$▼_M[CN=名前]$ // コメント [5]        ← 簡略形（⊕5+0 と同等）
```

| 記号 | 意味 |
|------|------|
| `⊕` | 展開状態 |
| `⊖` | 折畳状態 |
| `⊘` | ロック（操作不可） |
| `∞` | カウントしない（無制限） |
| `N+M` | カウント値 = N + 10^M（コンパクト表現） |
| `∞N+M` | N + 10^M でカウント停止済み |

クリックするたびにカウントが自動インクリメントされ、Markdownソースに書き戻される。

▼_M[CN=1008_相互参照] // 相互参照記法 v2.1（NEW）
// ⇄ {1009_固有番号}

開始膜の直後の行に `// ⇄ {}` で関連する膜を列挙する：

```
$▼_M[CN=1234_A]$ // Aの説明
// ⇄ {5678_B, 9012_C}

内容
$▲_M[CN=1234_A]$
```

**設計思想：**
- `⇄` = 双方向リンク
- `{}` で囲む = ツール・AIがパース可能な構造化リスト
- `grep "5678"` で参照先に即ジャンプ
- 目次に `⇄ {NNNN_name}` を併記 → 目次だけでリンクを辿れる
- 特別な仕組み不要。実身/仮身相当をプレーンテキストで実現
- どんなエディタ・AI・ツールでも等しく使える

**目次での記法：**
```
$▼_M[CN=0001_TOC]$
  $▼_M[CN=1234_A]$  ⇄ {5678_B, 9012_C}
  $▼_M[CN=5678_B]$  ⇄ {1234_A}
$▲_M[CN=0001_TOC]$
```
▲_M[CN=1008_相互参照]

// ⇄ {1008_相互参照, 1010_AIメモリ}

```
CN=NNNN_名前
例: CN=5831_FILE_HEADER
    CN=3658_insertTemplate
    CN=4471_mupInsertBookmark
```

**設計思想：**
- `NNNN` = ランダム4桁数字（名前との合成キーで衝突リスクを排除）
- `名前` = 人間が読める説明（日本語可）
- `grep "5831"` → 巨大ソースコードから1件だけヒット確定
- 4桁+名前の組合せ = 実質無限の固有値空間

**メリット：**
- 数字だけで検索 → 部分一致なし、1件確定
- セッション間引き継ぎの「コードの住所」
- AIとの協働効率を劇的に向上（体感100倍以上）

▼_M[CN=1013_記法変遷] // 記法バージョン変遷
| バージョン | 記法 | 備考 |
|-----------|------|------|
| v1.x | `\▼[CN=...]` | バックスラッシュ記法（初期） |
| v2.x | `$\▼[CN=...]$` | LaTeX保護記法（TinyMCE対策） |
| v3.0 | `$M▼[CN=...]$` | M記法（旧）`$M▼$` 形式 |
| v3.1 | `$▼_M[CN=...]$` | **▼_M記法（現行）** ▼先頭が折畳み方向を直感的に表す。`_M`はLaTeXサブスクリプト（Membrane）。`$`がKaTeXウィジェット境界として機能しTinyMCEが保護。 |

**`$▼_M[...]$` の設計思想：**
- `$...$` = LaTeX/KaTeX記法 → TinyMCE（WYSIWYG）がウィジェットとして保護 → 膜が壊れない
- Markdownモード：生テキストとしてそのまま表示・編集
- プレビュー：膜UI・しおりボタンとして描画
- WYSIWYG：KaTeXウィジェットとして完全保護

**旧記法からの移行：** メニュー `Fold Membrane → Migrate 旧記法 → $▼_M$ 記法 ⚡`
（`\▼`・`$\▼$`・`A▼`・`M▼`・`$M▼$` を自動で `$▼_M[...]$` 形式に一括変換）
